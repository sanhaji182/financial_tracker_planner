## Prompt

```
Baca context files: AGENTS.md, api-conventions.md (Transactions), database-schema.md (transactions, transaction_splits, transaction_attachments, categories, audit_logs), business-rules.md (bagian Transaction Rules dan Account Balance), ui-design-system.md (Table, Badge specs).

Implementasikan modul Transaksi lengkap:

═══ BACKEND ═══

**Models**: Transaction, TransactionSplit, TransactionAttachment, Category, AuditLog — sesuai DB schema

**Category Management:**
- GET /api/v1/categories → list semua (system + user custom)
- POST /api/v1/categories → create custom category
- PUT /api/v1/categories/:id → update (hanya custom, bukan system)
- DELETE /api/v1/categories/:id → soft delete (hanya custom)

**Transaction CRUD:**
- POST /api/v1/transactions → create
  - Input: date, amount, type (income/expense), account_id, category_id, description, notes
  - ATOMIC: insert transaction + update account balance dalam 1 DB transaction
  - Insert audit_log (action: create)
  - Invalidate dashboard cache di Redis
- GET /api/v1/transactions → list (paginated)
  - Filters: type, category_id, account_id, date_from, date_to, amount_min, amount_max, search (description), status
  - Sort: date (default desc), amount, created_at
  - Include: category name, account name
- GET /api/v1/transactions/:id → detail + splits + attachments + audit trail
- PUT /api/v1/transactions/:id → update
  - ATOMIC: reverse old balance impact → apply new impact
  - Insert audit_log (action: update, old_value, new_value)
- DELETE /api/v1/transactions/:id → soft delete
  - ATOMIC: reverse balance impact
  - Insert audit_log (action: delete)

**Audit Log Service:**
- CreateAuditLog(ctx, userID, entityType, entityID, action, oldValue, newValue) → error
- GetAuditLogs(ctx, entityType, entityID) → []AuditLog

═══ FRONTEND ═══

**Page: TransactionsPage.tsx** (/transactions)
- Filter bar di atas:
  - Date range picker (default: bulan ini)
  - Dropdown: Tipe (Semua/Pemasukan/Pengeluaran)
  - Dropdown: Kategori
  - Dropdown: Akun
  - Search input (debounced)
  - Tombol "Reset Filter"
- Summary row: Total Income | Total Expense | Net (bulan terpilih)
- Tabel transaksi:
  - Kolom: Tanggal | Deskripsi | Kategori (icon + nama) | Akun | Jumlah | Status
  - Warna jumlah: hijau income, merah expense
  - Klik row → detail drawer/modal
  - Pagination di bawah
- Tombol "+ Tambah Transaksi" (FAB di mobile, button di desktop)

**Modal: TransactionFormModal.tsx**
- Tabs: Pemasukan | Pengeluaran (switch type)
- Fields: Tanggal (datepicker), Jumlah (number, format Rp otomatis), Akun (select), Kategori (select with icon), Deskripsi, Catatan (optional)
- Upload bukti transaksi (drag & drop images)
- Button: Simpan (loading state)

**Detail: TransactionDetailDrawer.tsx**
- Semua info transaksi
- Attachments preview (thumbnail images)
- Audit trail timeline: "Dibuat oleh Owner pada 5 Jul 2026", "Diubah: kategori dari Makan ke Transport pada 6 Jul"
- Tombol: Edit, Hapus (konfirmasi)

**Category Management: CategoriesPage.tsx** (/settings/categories)
- List kategori expense dan income (grouped)
- Icon + warna + nama
- Add custom category
- Edit/delete custom categories (system categories read-only)

PENTING (dari business-rules.md):
- Saat create income: account.balance += amount
- Saat create expense: account.balance -= amount
- Saat update: reverse old → apply new (ATOMIC)
- Saat delete: reverse impact (ATOMIC)
- Semua perubahan HARUS masuk audit_log
```
