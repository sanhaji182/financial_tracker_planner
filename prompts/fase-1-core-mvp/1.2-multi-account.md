## Prompt

```
Baca context files: AGENTS.md, api-conventions.md (Accounts), database-schema.md (tabel accounts), ui-design-system.md.

Implementasikan modul Multi-Account Management:

═══ BACKEND ═══

Model: Account sesuai database schema (accounts table)

DTOs:
- CreateAccountRequest: name, type, bank_provider, account_number, initial_balance, currency, is_shared, is_emergency_fund, notes
- UpdateAccountRequest: name, bank_provider, is_active, is_shared, is_emergency_fund, notes
- AccountResponse: semua fields + formatted balance
- AccountSummaryResponse: total per tipe, grand total

Repository (account_repo.go):
- Create, GetByID, GetAllByUser, Update, SoftDelete
- GetSummaryByUser → aggregate saldo per tipe
- UpdateBalance(ctx, accountID, delta) → atomic balance update

Service (account_service.go):
- CreateAccount → set initial_balance, validasi tipe
- UpdateAccount → tidak boleh ubah tipe dan initial_balance
- DeleteAccount → soft delete, validasi tidak ada transaksi aktif
- GetAccounts → list semua akun user, sorted by sort_order
- GetAccountSummary → total per tipe + grand total
- GetAccountByID → detail akun

Handler (account_handler.go):
- GET    /api/v1/accounts → list
- GET    /api/v1/accounts/:id → detail
- POST   /api/v1/accounts → create
- PUT    /api/v1/accounts/:id → update
- DELETE /api/v1/accounts/:id → soft delete
- GET    /api/v1/accounts/summary → summary per tipe

Semua endpoint: protected, owner only.

═══ FRONTEND ═══

Page: AccountsPage.tsx (/accounts)
- Top summary cards: Total Kas | Total Investasi | Total Deposit | Grand Total
- Grid/list akun dengan card view
- Setiap card:
  - Icon sesuai tipe (🏦 bank, 📱 e-wallet, 💵 cash, 📈 investment, 🏧 deposit)
  - Nama akun
  - Bank/provider
  - Saldo (MoneyDisplay component, monospace)
  - Badge "Bersama" jika is_shared
  - Badge "Dana Darurat" jika is_emergency_fund
  - Menu dropdown: Edit, Archive, Delete
- Tombol "+ Tambah Akun" di top right
- Modal CreateAccountModal:
  - Input: nama, tipe (select), bank/provider, nomor rekening, saldo awal, mata uang, is_shared toggle, is_emergency_fund toggle, catatan
  - Validasi: nama wajib, saldo >= 0
  - Loading state saat submit
- Empty state: "Belum ada akun keuangan. Tambahkan rekening bank, e-wallet, atau kas tunai Anda."

Service (accounts.ts):
- getAccounts() → Account[]
- getAccount(id) → Account
- createAccount(data) → Account
- updateAccount(id, data) → Account
- deleteAccount(id) → void
- getAccountSummary() → AccountSummary

Hooks:
- useAccounts() → React Query hook for list
- useAccountSummary() → React Query hook for summary

PENTING:
- Icon dan warna berbeda per tipe akun
- Nomor rekening ditampilkan masked: ****1234
- Saldo menggunakan MoneyDisplay component (format Rp, monospace)
- Card hover effect subtle
```
