## Prompt

```
Baca context files: AGENTS.md, context/api-conventions.md (bagian Auth), context/database-schema.md (tabel users, refresh_tokens), context/architecture.md (Authentication Flow).

Implementasikan sistem Authentication lengkap:

═══ BACKEND (Golang) ═══

1. **Models** (/backend/internal/model/user.go):
   - User struct sesuai database schema
   - UserResponse (tanpa password_hash)

2. **DTOs** (/backend/internal/dto/auth.go):
   - RegisterRequest (email, password, name) + validation tags
   - LoginRequest (email, password) + validation
   - AuthResponse (access_token, refresh_token, user)
   - InviteSpouseRequest (email)
   - ChangePasswordRequest (old_password, new_password)

3. **Repository** (/backend/internal/repository/user_repo.go):
   - CreateUser(ctx, user) → User, error
   - GetUserByEmail(ctx, email) → User, error
   - GetUserByID(ctx, id) → User, error
   - UpdateUser(ctx, user) → error
   - CreateRefreshToken(ctx, token) → error
   - GetRefreshToken(ctx, tokenHash) → RefreshToken, error
   - RevokeRefreshToken(ctx, tokenHash) → error
   - RevokeAllUserTokens(ctx, userID) → error

4. **Service** (/backend/internal/service/auth_service.go):
   - Register(ctx, req) → AuthResponse, error
     - Validate email uniqueness
     - Hash password (bcrypt cost 12)
     - Create user with role 'owner'
     - Generate JWT pair
   - Login(ctx, req) → AuthResponse, error
     - Find user by email
     - Verify password
     - Update last_login_at
     - Generate JWT pair
   - RefreshToken(ctx, refreshToken) → AuthResponse, error
     - Verify refresh token validity
     - Check not revoked
     - Rotate: revoke old, create new pair
   - Logout(ctx, refreshToken) → error
     - Revoke refresh token
   - InviteSpouse(ctx, ownerID, req) → InviteLink, error
     - Only owner can invite
     - Generate unique invite token
     - Return invite link
   - RegisterSpouse(ctx, inviteToken, req) → AuthResponse, error
     - Validate invite token
     - Create user with role 'spouse_viewer', invited_by = ownerID
   - ChangePassword(ctx, userID, req) → error

5. **JWT Util** (/backend/internal/util/jwt.go):
   - GenerateAccessToken(userID, role, email) → string, error
   - GenerateRefreshToken() → string, error
   - ValidateAccessToken(tokenString) → Claims, error
   - HashToken(token) → string

6. **Middleware** (/backend/internal/middleware/auth.go):
   - AuthMiddleware() → gin.HandlerFunc
     - Extract Bearer token from Authorization header
     - Validate JWT
     - Set user_id, role, email in gin.Context
   - RoleMiddleware(roles ...string) → gin.HandlerFunc
     - Check if user role is in allowed roles
     - Return 403 if not

7. **Handler** (/backend/internal/handler/auth_handler.go):
   - POST /api/v1/auth/register
   - POST /api/v1/auth/login
   - POST /api/v1/auth/refresh
   - POST /api/v1/auth/logout (protected)
   - POST /api/v1/auth/invite-spouse (protected, owner only)
   - POST /api/v1/auth/register-spouse
   - PUT /api/v1/auth/change-password (protected)

═══ FRONTEND (React) ═══

1. **Auth Store** (/frontend/src/stores/authStore.ts):
   - State: user, accessToken, isAuthenticated, isLoading
   - Actions: setAuth, clearAuth, updateUser

2. **Auth Service** (/frontend/src/services/auth.ts):
   - login(email, password) → AuthResponse
   - register(email, password, name) → AuthResponse
   - refreshToken() → AuthResponse
   - logout() → void
   - inviteSpouse(email) → InviteLink
   - changePassword(oldPw, newPw) → void

3. **API Interceptor** (/frontend/src/services/api.ts):
   - Request interceptor: attach access token
   - Response interceptor: jika 401 → try refresh → retry original request
   - Jika refresh gagal → redirect ke login

4. **Pages**:
   - LoginPage.tsx:
     - Clean, premium design (bukan template generic)
     - Logo/app name di atas
     - Email + Password fields
     - "Ingat saya" checkbox
     - Login button (dengan loading state)
     - Link ke register
     - Error message yang jelas
   - RegisterPage.tsx:
     - Nama + Email + Password + Confirm Password
     - Password strength indicator
     - Register button (loading state)
   - InviteSpousePage.tsx (protected, owner only):
     - Input email pasangan
     - Generate invite link
     - Copy link button

5. **Protected Route** (/frontend/src/components/shared/ProtectedRoute.tsx):
   - Check isAuthenticated
   - Jika tidak → redirect ke /login
   - Optional: check role

6. **Router** (/frontend/src/App.tsx):
   - /login → LoginPage
   - /register → RegisterPage
   - /register-spouse/:token → RegisterSpousePage
   - /* (protected) → AppShell with nested routes

═══ UI REQUIREMENTS ═══
- Login page: centered card, subtle background gradient, app branding
- Form validation realtime (email format, password min 8 chars)
- Loading spinner pada tombol saat submit
- Toast notification untuk success/error
- Responsive: login page harus bagus di mobile
```
