## Prompt

```
Baca context: AGENTS.md, database-schema.md (assets, asset_valuations), api-conventions.md (Assets).

Implementasikan modul Aset:

═══ BACKEND ═══

Repository (asset_repo.go):
- Create, GetByID, GetAllByUser (filter by type, is_shared), Update, SoftDelete
- CreateValuation, GetValuationsByAsset
- GetSummaryByUser → total per tipe + grand total + shared vs pribadi

Service (asset_service.go):
- CreateAsset → jika linked_account_id ada, sync current_value dengan account balance
- UpdateAsset → update current_value, auto-create valuation entry
- DeleteAsset → soft delete
- AddValuation → insert ke asset_valuations + update current_value di assets
- GetAssets → list filtered
- GetAssetSummary → total aset per tipe, total shared, total pribadi

Handler (asset_handler.go):
- GET    /api/v1/assets → list (filter: type, is_shared)
- GET    /api/v1/assets/:id → detail + valuations
- POST   /api/v1/assets → create
- PUT    /api/v1/assets/:id → update
- DELETE /api/v1/assets/:id → soft delete
- POST   /api/v1/assets/:id/valuations → add valuation
- GET    /api/v1/assets/summary → summary

═══ FRONTEND ═══

Page: AssetsPage.tsx (/assets)
- Top summary: Total Aset | Liquid | Investasi | Properti | Kendaraan
- Grid cards grouped by tipe:
  - 💰 Tabungan & Rekening
  - 🏠 Properti
  - 🚗 Kendaraan
  - 📈 Investasi
  - 💵 Kas Tunai & E-wallet
  - 📦 Lainnya
- Card per aset: icon, nama, nilai (MoneyDisplay), badge Bersama/Pribadi, badge tipe
- Modal form: nama, tipe (select), nilai saat ini, nilai beli (optional), tanggal beli, linked account (select), is_shared toggle, catatan, metadata (flexible: alamat untuk properti, plat untuk kendaraan)
- Detail view:
  - Info lengkap
  - Chart: histori nilai (line chart dari asset_valuations)
  - Tombol "Update Nilai" → form input nilai baru
  - Gain/Loss: current_value vs purchase_value (jika ada)

Empty state: "Belum ada aset tercatat. Tambahkan properti, kendaraan, investasi, atau tabungan Anda."
```
