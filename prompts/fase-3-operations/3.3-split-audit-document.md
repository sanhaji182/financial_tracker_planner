## Prompt

```
Baca context: AGENTS.md, database-schema.md (transaction_splits, audit_logs, documents), business-rules.md (Split Transaction).

Implementasikan tiga fitur:

═══ 1. SPLIT TRANSACTION ═══

Backend:
- POST /api/v1/transactions/:id/split → {splits: [{category_id, amount, description}]}
  - Validasi: sum(split.amount) HARUS = transaction.amount
  - Set transaction.is_split = true
  - Budget tracking: per split category, bukan parent
- GET /api/v1/transactions/:id → include splits

Frontend:
- Button "Split" di detail transaksi
- Split form: add/remove rows, kategori select per row, amount per row
- Validation: total splits must equal transaction amount
- Visual: breakdown bar di detail view

═══ 2. AUDIT TRAIL ═══

Backend (sudah ada dari 1.3, enhance):
- GET /api/v1/audit-logs → global list (filter: entity_type, date_range, user)
- GET /api/v1/audit-logs/:entity_type/:entity_id → per entity
- Include: user name, timestamp, IP address
- Cover: transactions, debts, assets, bills, reconciliation, closing

Frontend:
- Per entity: timeline view di detail drawer
  - "Owner membuat transaksi — 5 Jul 2026 14:30"
  - "Kategori diubah dari Makan ke Transport — 6 Jul 2026 09:15"
  - Show old → new value diff
- AuditLogPage.tsx (/admin/audit-log) — global view
  - Tabel: Waktu | User | Entitas | Aksi | Detail
  - Filter by entity type, date range

═══ 3. DOCUMENT CENTER ═══

Backend:
- POST /api/v1/documents → upload file (multipart, max 10MB)
  - Save to storage (local/S3)
  - Insert record with linked_entity_type + linked_entity_id
- GET /api/v1/documents → list (filter: entity_type, tags)
- GET /api/v1/documents/:id/download → serve file
- DELETE /api/v1/documents/:id → soft delete + remove from storage

Frontend: DocumentCenterPage.tsx (/documents)
- Upload: drag & drop zone
- Grid/list view: thumbnail (images) or icon (PDF)
- Filter: by entity type, by tags
- Preview: lightbox for images, new tab for PDF
- Link documents to existing transactions/assets/debts via modal

Supported: PDF, JPG, PNG, JPEG (max 10MB)
```
