## Prompt

```
Baca context: AGENTS.md, database-schema.md (monthly_closings), business-rules.md (Reconciliation + Monthly Closing Snapshot — ikuti format snapshot PERSIS).

Implementasikan Reconciliation dan Monthly Closing:

═══ RECONCILIATION ═══

Backend:
- POST /api/v1/reconciliation/start → {account_id, actual_balance, date}
  - Calculate difference = app_balance - actual_balance
  - Return: difference, unmatched transactions, suggestions
- POST /api/v1/reconciliation/confirm → mark all transactions as reconciled

Frontend: ReconciliationWizard (step-by-step)
1. Pilih akun
2. Input saldo nyata per tanggal
3. Show selisih + transaksi unmatched
4. User: add missing, edit wrong, atau confirm
5. Done → semua marked reconciled

═══ MONTHLY CLOSING ═══

Backend:
- POST /api/v1/monthly-closing/generate?month=2026-07
  - Generate snapshot PERSIS sesuai business-rules.md format
  - Save to monthly_closings (immutable after confirmed)
  - Generate month-over-month comparison
- GET /api/v1/monthly-closing → list all closings
- GET /api/v1/monthly-closing/:month → detail + comparison vs previous

Frontend: MonthlyClosingPage.tsx (/closing)
- List closings by month
- Detail report: professional layout, printable
  - Net Worth, Assets, Debts, Income vs Expense
  - Budget vs Actual table
  - Goal progress
  - Month-over-month changes (↑/↓ with delta)
  - Charts: income vs expense bar, asset composition pie
- Button "Generate Closing" (disabled if already exists for month)
- Button "Export PDF"
- Immutable label after confirmed
```
