## Prompt

```
Baca context: AGENTS.md, database-schema.md (alerts), business-rules.md (Alert Priority — tabel prioritas).

Implementasikan Alert Center:

═══ BACKEND ═══

Alert Generator Service (cron — setiap 6 jam):
- Scan dan generate alerts sesuai prioritas di business-rules.md:
  1. Tagihan overdue (danger)
  2. Forecast saldo < 0 (danger)
  3. EF < 3 bulan (danger)
  4. Tagihan H-1 (warning)
  5. Budget > 100% (warning)
  6. Tagihan H-3 (warning)
  7. Budget > 80% (warning)
  8. Subscription renewal (info)
  9. Parsing perlu review (info)
  10. Insight tersedia (info)
- Avoid duplicates: jangan buat alert yang sudah ada dan belum di-dismiss
- Critical alerts (danger) → kirim ke Telegram bot

Telegram Bot Integration:
- Service: telegram_service.go
- Send message via Telegram Bot API
- Format: "⚠️ [title]\n[message]\n[action hint]"
- Config: bot_token + chat_id dari env

Handler:
- GET /api/v1/alerts → list (paginated, filter: type, severity, is_read)
- PUT /api/v1/alerts/:id/read
- PUT /api/v1/alerts/mark-all-read
- DELETE /api/v1/alerts/:id → dismiss
- GET /api/v1/alerts/unread-count → for badge

═══ FRONTEND ═══

AlertCenterPage.tsx (/alerts):
- Inbox-style list
- Filter: All | Unread | By severity | By type
- Per alert: severity icon + title + message + time ago + action button
- Mark as read (click or explicit)
- "Mark all as read" button
- Dismiss button

TopBar integration:
- Bell icon 🔔 with unread count badge
- Dropdown on click: top 5 recent alerts
- "Lihat semua →" link

Dashboard integration:
- Alert summary panel (top 5 most important)

Colors: info=biru, warning=kuning, danger=merah
```
