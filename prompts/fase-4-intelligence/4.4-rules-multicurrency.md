## Prompt

```
Baca context: AGENTS.md, database-schema.md (automation_rules, currencies).

═══ 1. RULE-BASED AUTO ACTIONS ═══

Backend:
- CRUD /api/v1/automation-rules
- Rule Engine Service (cron — daily):
  - Load all active rules
  - Evaluate each rule against current data
  - Execute action if condition met
  - Log execution (trigger_count++, last_triggered_at)

Trigger types:
- balance_below: condition={account_id, threshold} → alert if balance < threshold
- bill_due_soon: condition={days_before} → reminder for bills due within N days
- budget_exceeded: condition={percentage: 80} → alert when budget > X%
- recurring_transaction: condition={amount, category_id, account_id, frequency, day_of_month} → auto-create transaction

Action types:
- send_alert: create in-app alert
- send_telegram: send Telegram message
- create_transaction: auto-create recurring transaction

Frontend: AutomationPage.tsx (/settings/automation)
- List rules: name, trigger, condition summary, action, active toggle
- Rule builder: select trigger → set conditions → select action → save
- Execution history per rule
- Toggle active/inactive

═══ 2. MULTI-CURRENCY ═══

Backend:
- currencies table already seeded (IDR, USD, SGD, EUR)
- Any account/asset/debt can have currency != IDR
- Conversion to IDR for net_worth/total calculations using exchange_rate_to_idr
- PUT /api/v1/currencies/:code → update exchange rate
- GET /api/v1/currencies → list

Frontend:
- Currency selector in account/asset/debt forms
- Display: "S$ 5,000 (≈ Rp 61.500.000)" — original + IDR equivalent
- Settings page: manage exchange rates (manual update)
- Dashboard calculations: all converted to IDR
```
