# Prompt 4.2 — Monthly Insight Engine

> **Fase**: 4 (Intelligence) | **Prasyarat**: Fase 1-2 + 3 bulan data
> **Output**: Automated monthly insights — spending analysis, trends, recommendations

---

## Context Files
- `../AGENTS.md`, `../context/database-schema.md` (monthly_insights), `../context/business-rules.md`

---

## Prompt

```
Baca context: AGENTS.md, database-schema.md (monthly_insights).

Implementasikan Monthly Insight Engine:

═══ BACKEND ═══

Service (insight_service.go):
- GenerateInsights(ctx, userID, month) → create/update insights
- Insight types (auto-calculated):
  1. spending_increase: per kategori vs rata-rata 3 bulan — "Makan naik 23% dari rata-rata"
  2. top_categories: top 3 pengeluaran terbesar — "Makan Rp 5jt, Transport Rp 3jt, Belanja Rp 2.5jt"
  3. subscription_change: total subscription cost berubah — "Subscription naik Rp 150rb"
  4. cashflow_risk: tanggal spending spike — "Pengeluaran terbesar di minggu ke-2"
  5. networth_trend: net worth bulan ini vs bulan lalu — "Net worth naik 2.5%"
  6. recommendation: saran bulan berikutnya — "Kurangi belanja online, sudah over budget 3 bulan berturut"
- Severity: positive (green), neutral (blue), negative (red)

Handler:
- GET /api/v1/insights?month=2026-07
- POST /api/v1/insights/generate?month=2026-07 (manual trigger)

═══ FRONTEND ═══

InsightsPage.tsx (/insights):
- Month picker
- Card list per insight:
  - Severity icon (✅ positive, ℹ️ neutral, ⚠️ negative)
  - Title + description
  - Supporting data (mini chart or number)
  - Expandable detail
- Dashboard integration: top 3 insights in summary
```
