## Prompt

```
Baca context: AGENTS.md, database-schema.md (scenarios), business-rules.md (What-If Impact Calculation — ikuti formula).

Implementasikan What-If Scenario Planner:

═══ BACKEND ═══

Service (scenario_service.go):
- SimulateScenario(ctx, userID, changes []ScenarioChange) → ScenarioResult
- Change types:
  - extra_debt_payment: {debt_id, monthly_extra_amount}
  - income_change: {percentage} (positive or negative)
  - large_purchase: {amount, month}
  - investment_increase: {monthly_amount}
  - add_subscription: {monthly_amount}
  - remove_expense: {category_id, monthly_amount}

- Calculate impact per business-rules.md:
  - base_state: current forecast, debts, EF coverage, cash_runway
  - scenario_state: apply all changes
  - impact: scenario - base (per metric)

- SaveScenario(ctx, userID, name, changes, result) → save for reference
- GetScenarios(ctx, userID) → list saved

Handler:
- POST /api/v1/scenarios/simulate → simulation result (not saved)
- POST /api/v1/scenarios → save scenario
- GET /api/v1/scenarios → list saved
- DELETE /api/v1/scenarios/:id

═══ FRONTEND ═══

ScenarioPage.tsx (/scenarios):

Scenario Builder:
- Dynamic form: "Tambah perubahan" → pilih tipe → isi parameter
- Stack multiple changes
- Button: "Simulasikan"

Result view: side-by-side comparison
┌─────────────────┬──────────┬──────────┬──────────┐
│ Metrik          │ Saat Ini │ Skenario │ Dampak   │
├─────────────────┼──────────┼──────────┼──────────┤
│ Saldo akhir     │ Rp 8.5jt │ Rp 5.5jt │ -Rp 3jt 🔴│
│ Outstanding     │ Rp 200jt │ Rp 195jt │ -Rp 5jt 🟢│
│ EF Coverage     │ 4.2 bln  │ 4.2 bln  │ 0 ⚪     │
│ Cash Runway     │ 2.5 bln  │ 1.8 bln  │ -0.7 🟡  │
└─────────────────┴──────────┴──────────┴──────────┘

- Bar chart comparison per metric
- Highlight positive (green) and negative (red) impacts
- "Simpan Skenario" button → name and save
- "Buat Ulang" → reset form
- List saved scenarios with quick-load
```
