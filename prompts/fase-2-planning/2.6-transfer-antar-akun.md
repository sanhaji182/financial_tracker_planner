# Prompt 2.6 — Transfer Antar Akun

> **Fase**: 2 (Planning) | **Prasyarat**: Prompt 1.2 + 1.3 selesai
> **Output**: Inter-account transfer — atomic balance updates, not counted as income/expense

---

## Context Files
- `../AGENTS.md`, `../context/business-rules.md` (Transaction Rules — Transfer), `../context/database-schema.md` (transactions — type transfer)

---

## Prompt

```
Baca context: AGENTS.md, business-rules.md (Transfer rules), database-schema.md (transactions — target_account_id).

Implementasikan Transfer Antar Akun:

═══ BACKEND ═══

Service (transfer_service.go):
- CreateTransfer(source_account_id, target_account_id, amount, date, notes):
  - Validasi: source != target, amount > 0, amount <= source.balance
  - ATOMIC DB transaction:
    1. source_account.balance -= amount
    2. target_account.balance += amount
    3. Insert transaction (type='transfer', account_id=source, target_account_id=target)
    4. Insert audit_log
  - Transfer TIDAK dihitung sebagai income/expense
  - Transfer TIDAK masuk budget tracking

Handler:
- POST /api/v1/transfers → create transfer
- GET /api/v1/transfers → list transfers (filter by date, accounts)

═══ FRONTEND ═══

TransferModal.tsx (accessible dari halaman Accounts dan Transactions):
- Select akun sumber (show saldo): [BCA — Rp 15.000.000 ▼]
- Select akun tujuan: [GoPay — Rp 500.000 ▼]
- Input jumlah (validasi ≤ saldo sumber)
- Tanggal
- Catatan (optional)
- Button: "Transfer"

Di list transaksi:
- Transfer ditampilkan dengan icon panah dua arah ↔️
- Warna biru (bukan hijau/merah)
- Label: "Transfer: BCA → GoPay"
- Badge "Transfer"

Quick transfer shortcuts di AccountsPage: "Transfer ke..." dropdown per akun
```
