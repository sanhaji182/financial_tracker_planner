## Prompt

```
Baca context: AGENTS.md, database-schema.md (bills, bill_payments), api-conventions.md (Bills).

Implementasikan modul Kalender Tagihan & Recurring Bills:

═══ BACKEND ═══

Service (bill_service.go):
- CreateBill → set next_due_date berdasarkan frequency dan due_day
- UpdateBill → recalculate next_due_date jika frequency/due_day berubah
- PayBill → buat bill_payment + transaksi expense linked + update status
  - Jika jumlah bayar < bill amount → is_partial=true, remaining_amount = bill.amount - paid
  - Jika jumlah bayar >= bill amount → status='paid', generate next occurrence untuk recurring
- Auto-update status cron: setiap hari, cek bills WHERE next_due_date < today AND status='unpaid' → set status='overdue'
- Recurring generation: setelah paid, buat bill entry baru untuk periode berikutnya
- GetUpcoming(days) → bills WHERE next_due_date BETWEEN today AND today+N days
- GetMonthlyCommitment(month) → total bills dalam bulan tertentu

Handler:
- GET /api/v1/bills → list (filter: status, month)
- GET /api/v1/bills/:id → detail + payment history
- POST /api/v1/bills → create
- PUT /api/v1/bills/:id → update
- DELETE /api/v1/bills/:id → soft delete
- POST /api/v1/bills/:id/payments → pay (full or partial)
- GET /api/v1/bills/upcoming?days=7 → upcoming bills
- GET /api/v1/bills/monthly-commitment?month=2026-07 → total komitmen

═══ FRONTEND ═══

Page: BillsPage.tsx (/bills)
- Tabs: Daftar | Kalender
- Summary bar: Total Bulan Ini | Sudah Bayar ✅ | Belum ⏳ | Overdue 🔴

Tab "Daftar":
- Tabel tagihan: Nama | Jumlah | Jatuh Tempo | Frekuensi | Akun | Status
- Status badges: paid ✅ / unpaid ⏳ / overdue 🔴
- Quick action: "Bayar" button per row
- Filter: status, month picker

Tab "Kalender":
- Month calendar view
- Dots/badges pada tanggal yang ada tagihan
- Klik tanggal → popup list tagihan hari itu
- Color coded: hijau (paid), kuning (unpaid), merah (overdue)

Modal form: nama, jumlah, frekuensi (select: bulanan/tahunan/quarterly/weekly/custom), due_day/due_date, akun sumber, kategori, auto_remind toggle, reminder_days_before, catatan

Payment modal: jumlah (default full), tanggal, catatan
- Jika partial → show remaining

Detail view: histori pembayaran, remaining balance (jika partial)

Integrasi dashboard: upcoming bills 7 hari widget sudah mengambil dari endpoint ini
```
