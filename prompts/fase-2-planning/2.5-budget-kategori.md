## Prompt

```
Baca context: AGENTS.md, database-schema.md (budgets), business-rules.md (Budget Rules), api-conventions.md (Budgets).

Implementasikan Budget per Kategori:

═══ BACKEND ═══

Service (budget_service.go):
- SetBudget(category_id, month, amount) → upsert
- GetBudgets(month) → list + realisasi per kategori
  - Realisasi = SUM(transactions.amount) WHERE category_id=X AND month=Y AND type='expense'
  - Include: used_percentage, status (on_track/attention/almost/over)
- GetBudgetSummary(month) → total budget, total spent, remaining, categories over
- CopyFromPreviousMonth(month) → copy budgets dari bulan sebelumnya
- Warning flags sesuai business-rules.md: <60% on_track, 60-80% attention, 80-100% almost, >100% over

Handler:
- GET /api/v1/budgets?month=2026-07
- POST /api/v1/budgets (category_id, month, amount)
- PUT /api/v1/budgets/:id (amount)
- DELETE /api/v1/budgets/:id
- POST /api/v1/budgets/copy?from=2026-06&to=2026-07
- GET /api/v1/budgets/summary?month=2026-07

═══ FRONTEND ═══

Page: BudgetsPage.tsx (/budgets)
- Month picker di header
- Summary: Total Budget Rp 20jt | Spent Rp 14jt | Remaining Rp 6jt
- Per kategori:
  - Nama kategori (icon + warna)
  - Progress bar: hijau <60%, kuning 60-80%, oranye 80-100%, merah >100%
  - "Rp 4.800.000 / Rp 5.000.000 (96%)"
  - Quick edit amount (inline)
- Horizontal bar chart: budget vs actual per kategori
- Button "Copy dari bulan lalu"
- Button "+ Set Budget Baru" → select kategori, input jumlah
- Warning badges juga muncul di dashboard

Empty state: "Belum ada budget bulan ini. Set anggaran per kategori untuk mengontrol pengeluaran."
```
