## Prompt

```
Baca context: AGENTS.md (aturan AI: opsional, berlabel, bisa dimatikan).

Implementasikan AI Enhancement layer — SEMUA fitur ini OPSIONAL dan bisa di-toggle on/off:

═══ KONFIGURASI ═══

Backend:
- Settings table/config: ai_enabled (bool), ai_provider (openai/anthropic/local), ai_model
- API key stored in Vaultwarden (vault_references)
- GET /api/v1/settings/ai → current config
- PUT /api/v1/settings/ai → update config

Frontend: SettingsPage.tsx (/settings/ai)
- Toggle: "Aktifkan Fitur AI" (master switch)
- Provider selector: OpenAI / Anthropic / Local LLM
- Model selector
- API key input (saved to vault)
- Per-feature toggles: OCR escalation, auto-categorize, advisor, anomaly

═══ 1. HYBRID OCR ESCALATION ═══

Worker enhancement:
- POST /ai/enhance-ocr → receive OCR result + image
  - If overall_confidence < 0.7 AND ai_enabled:
    - Send image + OCR result to LLM
    - Prompt: "This receipt was OCR'd with these results: [data]. Please correct any errors."
    - Merge LLM corrections
    - Mark source='ai_enhanced'

═══ 2. AUTO CATEGORIZATION ═══

Backend:
- When new transaction created (manual or parsed):
  - If ai_enabled → send description + amount + merchant to Worker
- Worker: POST /ai/categorize → {description, amount, merchant, available_categories[]}
  - LLM suggests category_id + confidence
  - Return suggestion (user can accept/reject)
- Learning: save accepted categorizations for pattern matching

Frontend:
- On transaction form: "🤖 Saran: Makan & Minum (85%)" → accept/reject button

═══ 3. ADVISOR CHAT ═══

Backend/Worker:
- POST /ai/chat → {message, context: dashboard_summary}
  - Feed LLM with: net worth, cash, debts, forecast, EF status, budget status
  - System prompt: "You are a personal finance advisor. Answer in Bahasa Indonesia. Be specific with numbers. Always explain your reasoning."
  - Stream response

Frontend: AdvisorChat (sidebar drawer or floating button)
- Chat interface: message bubbles
- Pre-built questions: "Bagaimana kondisi keuangan saya?", "Apakah saya bisa beli X?", "Tips hemat bulan ini?"
- Responses labeled: "🤖 Saran AI — bukan nasihat keuangan profesional"
- Explainable: every suggestion has "Alasan: ..."

═══ 4. ANOMALY DETECTION ═══

Worker:
- POST /ai/detect-anomaly → {recent_transactions, category_averages}
  - Detect: amount >> 2× category average
  - Detect: spending spike in specific category
  - Return: list of anomalies

Backend:
- Run anomaly check weekly (cron) or on-demand
- Create alerts for detected anomalies

CRITICAL RULES:
- ❌ SEMUA fitur AI HARUS bisa dimatikan tanpa mengganggu fitur inti
- ❌ Jika AI dimatikan, aplikasi HARUS tetap 100% fungsional
- ✅ Setiap output AI HARUS berlabel "🤖 Saran AI"
- ✅ Setiap saran HARUS ada penjelasan "mengapa"
```
