## Prompt

```
Baca context: AGENTS.md, architecture.md (Transaction Creation OCR data flow).

Implementasikan OCR dan PDF Parser di Python Worker:

═══ WORKER (Python FastAPI) ═══

1. POST /ocr/receipt → multipart image upload
   - OCR engine: Tesseract (via pytesseract)
   - Pre-processing: grayscale, contrast enhancement, deskew (Pillow)
   - Extract: merchant_name, date, items[], total, payment_method
   - Confidence score per field (0.0-1.0)
   - Response: {parsed_data, overall_confidence, needs_review: bool}
   - needs_review = true if confidence < 0.7

2. POST /parse/pdf-statement → multipart PDF upload
   - Parser: pdfplumber
   - Extract: list of transactions [{date, description, debit, credit, balance}]
   - Template-based: detect bank format (BCA, Mandiri, BNI, BRI) by header patterns
   - Response: {bank_detected, period, transactions[], confidence}

3. Health: GET /health

═══ BACKEND (Golang) ═══

Handler: POST /api/v1/transactions/upload
- Accept multipart file
- Save to storage
- Determine type (image → OCR, PDF → parse)
- Forward to worker via HTTP
- Save result as draft transactions (status='pending_review')
- Return parsed data for frontend review

Service:
- UploadAndParse(ctx, file) → ParsedResult
- ConfirmParsedTransaction(ctx, transactionID, userEdits) → Transaction
  - Apply user corrections
  - Set status='confirmed', source='ocr' or 'pdf_parse'
  - Update account balance

═══ FRONTEND ═══

UploadPage.tsx (/transactions/upload):
- Drag & drop zone (accept: image/*, application/pdf)
- Loading: "Memproses dokumen..."
- Review screen:
  - Side-by-side: original image/PDF preview | parsed result form
  - Editable fields: merchant, date, amount, category, account
  - Fields with low confidence: highlighted yellow border + "Confidence: 65%"
  - Auto-suggest category based on merchant name (if available)
  - Buttons: "Konfirmasi & Simpan" | "Edit & Simpan" | "Batalkan"

In transaction list:
- Badge: "📷 Dari OCR" or "📄 Dari PDF" (source indicator)
- Filter: source=ocr, source=pdf_parse

PENTING:
- OCR HARUS berfungsi tanpa AI/LLM (Tesseract only)
- Ini adalah fitur BASELINE, bukan AI feature
- LLM escalation ada di prompt 5.2
```
